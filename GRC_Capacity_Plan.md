# Capacity Plan: Solo GRC Analyst Project

**Project:** End-to-end GRC build-out for a tech / consulting firm
**Owner:** 1 person (you) — GRC Analyst
**Period:** Ongoing, phased delivery
**Availability:** Full-time (~40 hrs/week), realistic productive capacity ~32 hrs/week after meetings, learning, and admin

---

## Project Scope

Build a complete GRC program from scratch for a tech/consulting firm, covering:

1. Compliance framework (controls + policies)
2. Audit system (gap assessment)
3. GRC tool implementation
4. Security assessment and testing

Phased sequentially so each phase deepens skill and produces a portfolio artifact.

---

## Phased Plan

### Phase 1 — Compliance Framework (8–10 weeks)

**Goal:** Pick a framework, map it to the firm, and document policies.

| Workstream | Hours | Notes |
|------------|-------|-------|
| Framework selection (NIST CSF, ISO 27001, SOC 2) | 20 | Compare, pick one to anchor on |
| Control mapping to firm's environment | 80 | Identify in-scope systems, data, vendors |
| Policy authoring (10–15 core policies) | 100 | Acceptable use, access control, incident response, etc. |
| Stakeholder review and revisions | 40 | Walk leadership through draft |
| **Phase total** | **240 hrs** | ~7.5 weeks at 32 hrs/wk |

**Deliverable:** Compliance framework doc + policy library
**Skills built:** Framework literacy, policy writing, control mapping

---

### Phase 2 — Audit System (6–8 weeks)

**Goal:** Assess current state against the framework, find gaps.

| Workstream | Hours | Notes |
|------------|-------|-------|
| Audit scope and methodology | 16 | Define what's in/out |
| Evidence collection | 80 | Interviews, screenshots, configs, logs |
| Gap analysis | 60 | Map findings to control failures |
| Audit report and recommendations | 40 | Ranked by risk and effort |
| **Phase total** | **196 hrs** | ~6 weeks at 32 hrs/wk |

**Deliverable:** Audit report with risk-ranked findings
**Skills built:** Evidence gathering, gap analysis, audit reporting

---

### Phase 3 — GRC Tool Implementation (8–12 weeks)

**Goal:** Select, configure, and roll out a GRC platform.

| Workstream | Hours | Notes |
|------------|-------|-------|
| Tool evaluation (Vanta, Drata, Hyperproof, Archer) | 40 | Demo, pricing, fit |
| Procurement and onboarding | 24 | Contract, kickoff |
| Configuration (controls, policies, workflows) | 120 | Wire framework into the tool |
| Integrations (cloud, IAM, ticketing) | 60 | Auto-evidence collection |
| User training and rollout | 40 | Stakeholder enablement |
| **Phase total** | **284 hrs** | ~9 weeks at 32 hrs/wk |

**Deliverable:** Live GRC tool with controls and integrations configured
**Skills built:** Tool admin, integrations, change management

---

### Phase 4 — Security Assessment and Testing (6–8 weeks)

**Goal:** Validate controls actually work; find technical weaknesses.

| Workstream | Hours | Notes |
|------------|-------|-------|
| Threat modeling | 32 | STRIDE or attack-tree approach |
| Vulnerability assessment | 60 | Scanning, config review |
| Penetration test (coordinate or run) | 60 | Internal or coordinate vendor |
| Remediation plan | 32 | Risk-ranked fixes with owners |
| **Phase total** | **184 hrs** | ~6 weeks at 32 hrs/wk |

**Deliverable:** Security assessment report + remediation plan
**Skills built:** Threat modeling, vuln assessment, technical security

## Bottlenecks and Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| Solo execution — no peer review | Quality drift | Schedule monthly review with mentor or stakeholder |
| Phase 3 depends on tool budget approval | Schedule slip | Start free-tier eval in Phase 2 |
| Phase 4 needs technical depth (pen testing) | Skill gap | Take a course parallel to Phase 1–2 |
| Burnout from solo full-time | Quality + timeline | Hard-cap at 32 productive hrs/wk |

---

## Scenarios

| Scenario | Outcome |
|----------|---------|
| **Do all phases solo, sequential** | ~7 months, full skill build, strong portfolio |
| **Parallelize Phase 1 + tool eval (Phase 3 prep)** | Saves 2–3 weeks but increases context-switching |
| **Hire/contract a pen tester for Phase 4** | Saves 4 weeks, frees you for remediation work |
| **Descope Phase 4** | Cuts ~6 weeks but weakens portfolio depth |

---

## Recommendations

1. **Lock in 32 hrs/week as your real capacity.** Don't plan to 40 — you'll burn out and the work suffers.
2. **Treat each phase as a portfolio artifact.** End each phase with a polished deliverable you can show in interviews.
3. **Run phases sequentially for now.** You said you want to learn deeply — sequential gives you that.
4. **Schedule a monthly self-audit.** Are you on pace? What's blocking you? Adjust the plan.
5. **Document decisions as you go.** Future-you (and interviewers) will want to know *why* you picked NIST over ISO, Vanta over Drata, etc.
